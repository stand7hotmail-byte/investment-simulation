# Specification: シミュレーションの高度化とポートフォリオ比較機能の実装

## 概要
効率的フロンティア画面において、個別資産やカスタム配分の可視化を行い、グラフから直接ポートフォリオを保存・シミュレーションへ連携できるフローを構築する。また、複数のポートフォリオを同時に比較できる機能を追加し、積立シミュレーションの計算ロジックを適正化する。

## 機能要件

### 1. Efficient Frontier Analysis の拡張
- **個別資産のプロット表示:** 選択された各資産の単体でのリスク（ボラティリティ）と期待リターンの位置をグラフ上に表示し、ラベルを付与する。
- **カスタムポートフォリオのプロット:** ユーザーが資産ごとの配分比率（%）を手動入力できる UI を提供し、その配分に基づくリスク・リターンの点をグラフ上にリアルタイムで表示する。
- **グラフからの保存機能:** 
    - 効率的フロンティア上の任意の点、またはカスタムプロット点をクリックして選択できる。
    - 選択した配分に名前を付けてポートフォリオとして保存できる。
    - 保存されたポートフォリオは、積立シミュレーションの選択肢に即座に現れる。

### 2. シミュレーションと比較機能
- **複数ポートフォリオの選択:** 積立シミュレーションの「Select Portfolio」をマルチセレクト形式に変更し、複数の保存済みポートフォリオを同時に選択可能にする。
- **比較チャートの表示:** 選択されたすべてのポートフォリオについて、将来の資産推移（Projected Final Value）を一つの折れ線グラフに重ねて表示し、凡例で区別できるようにする。

### 3. 計算ロジックの修正
- **Projected Final Value の正常化:** 現在「上昇が急すぎる」と感じられる計算式を修正する。具体的には、期待リターンの年利から月利への換算方法（複利考慮の有無）や、積立タイミング（期首・期末）の整合性を確認し、一般的な金融計算モデルに合わせる。

## 非機能要件
- **UI の応答性:** カスタム配分の入力やグラフのクリック操作がスムーズに行えること。
- **データ整合性:** 保存したポートフォリオがすべてのシミュレーション機能で一貫して利用できること。

## 受入条件
- 効率的フロンティア上に個別資産の点が表示されていること。
- ユーザーが入力した任意の配分がグラフ上に点としてプロットされること。
- グラフ上の点をクリックして保存し、シミュレーションのプルダウン/リストに反映されること。
- 複数のポートフォリオを選択して、成長曲線を一つのグラフで比較できること。
- 積立シミュレーションの予測値が、妥当な（急激すぎない）数値になっていること。
