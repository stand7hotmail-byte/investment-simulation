# Specification: 資産ユニバースの拡大とフィルタリング機能の実装

## 概要
投資シミュレーションの精度と利便性を高めるため、現在数種類に限定されている資産データを、国内外のETF、投資信託（ETF含む）、暗号資産、コモディティ、リートへと大幅に拡大する。また、大量の資産から目的のものを容易に選択できるよう、フロントエンドにアセットクラスによるフィルタリング機能を導入する。

## 機能要件
1.  **シードデータの拡充:**
    - `backend/app/seed_assets.py` または新規シードスクリプトを拡張し、以下のカテゴリの主要な資産（ティッカーシンボル）を追加する。
        - **国内外ETF:** VTI, VOO, VYM, 1306, 2558, 1557 等
        - **投資信託/指数:** 日経平均、S&P500、eMAXIS Slim等の投資信託に対応するETF
        - **暗号資産:** BTC-USD, ETH-USD
        - **コモディティ:** GLD（金）, SLV（銀）, DBC（商品総合）
        - **リート:** IYR（米国リート）, 1343（東証リート指数）
2.  **過去データの自動収集:**
    - 追加されたすべての資産について、Yahoo Finance API を使用して可能な限り長期間（最大20年分）の過去価格データを一括取得する。
3.  **フロントエンド UI の拡張 (Asset Selector):**
    - 資産選択画面に「アセットクラス（株式、債券、コモディティ等）」によるフィルタリング・タブまたはドロップダウンを追加する。
    - 選択されたフィルタに基づき、表示される資産リストを動的に切り替える。

## 非機能要件
- **データ取得の堅牢性:** 多数の資産を一括で取得する際、レート制限やエラーに対処できるよう、リトライ処理やエラーログを適切に管理する。
- **UIパフォーマンス:** 資産数が増加しても、リストのレンダリングやフィルタリングが遅延しないように最適化する。

## 受入条件
- データベースに定義されたすべての新規資産クラスのデータが正常に登録されていること。
- すべての新規資産に対して、過去データの収集スクリプトが正常に完了し、チャートが表示されること。
- フロントエンドの資産選択画面で、アセットクラスによるフィルタリングが正しく動作すること。

## 範囲外 (Out of Scope)
- リアルタイムでの価格更新（日次バッチ処理を前提とする）。
- ユーザーによる任意のティッカーシンボルの手動追加機能（今回は定義済みリストの拡充のみ）。
