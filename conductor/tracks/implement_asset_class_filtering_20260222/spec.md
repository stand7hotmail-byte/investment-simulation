# アセットクラス別フィルタリング機能の仕様

## 1. 概要

ユーザーが資産選択時に、アセットクラス（例: 株式、債券、コモディティ、REIT、暗号資産など）に基づいて利用可能な資産をフィルタリングできる機能を提供します。これにより、ユーザーは特定の投資戦略に合致する資産を容易に選択し、シミュレーションの精度と利便性を向上させることができます。

## 2. 機能要件

- **アセットクラス選択UI**:
    - 資産選択インターフェース内に、利用可能なアセットクラスのリストを表示するUI要素（例: チェックボックス、ドロップダウン、タグリスト）を設ける。
    - 複数のアセットクラスを選択できるようにする。
- **リアルタイムフィルタリング**:
    - ユーザーがアセットクラスを選択・解除するたびに、利用可能な資産リストがリアルタイムで更新される。
    - フィルタリングは、効率的フロンティアシミュレーションの資産選択コンポーネントに適用される。
- **フィルタリングのリセット**:
    - 選択されたすべてのアセットクラスフィルタを解除し、すべての資産を表示する機能を提供する。
- **永続性 (オプション)**:
    - ユーザーが選択したアセットクラスフィルタの状態をローカルストレージに保存し、次回訪問時もその状態を復元できるようにする（スコープ外とすることも可能）。

## 3. 技術的要件

- **バックエンド**:
    - `AssetData` モデル（`backend/app/models.py`）に `asset_class` フィールドが存在することを利用する。
    - 必要に応じて、ユニークなアセットクラスのリストを取得するAPIエンドポイントを追加する。
- **フロントエンド**:
    - `frontend/src/components/simulation/AssetSelector.tsx` を修正し、フィルタリングUIを統合する。
    - `useAssets` フック（`frontend/src/hooks/useAssets.ts`）を拡張し、アセットクラスフィルタリングをサポートする。
    - 選択されたアセットクラスの状態を管理するためにZustandストア（`frontend/src/store/useSimulationStore.ts`）を利用する。

## 4. UI/UXの考慮事項

- アセットクラスリストは直感的で操作しやすいものであること。
- フィルタリングの適用状況が明確にユーザーに伝わること。
- フィルタリングによって表示される資産が0になった場合、その旨をユーザーに通知すること。

## 5. 影響範囲

- `frontend/src/components/simulation/AssetSelector.tsx`
- `frontend/src/hooks/useAssets.ts`
- `frontend/src/store/useSimulationStore.ts`
- `backend/app/main.py` (必要であれば新しいAPIエンドポイントの追加)
- `backend/app/crud.py` (必要であればアセットクラスでフィルタリングする機能の追加)
