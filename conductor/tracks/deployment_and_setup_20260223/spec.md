# Specification: デプロイ・本番環境構築 (Deployment and Production Setup)

## 1. Overview (概要)
投資シミュレーションアプリケーションを本番環境にデプロイし、一般公開可能な状態にする。フロントエンドは Vercel、バックエンドとデータベースは Railway を使用し、自動化されたデプロイプロセスを構築する。

## 2. Functional Requirements (機能要件)
- **フロントエンドデプロイ (Vercel):**
    - Next.js アプリケーションを Vercel にデプロイする。
    - `vercel env` を使用して、必要な環境変数（API URL, Supabase 認証情報など）を設定する。
- **バックエンドデプロイ (Railway):**
    - FastAPI アプリケーションを Railway にデプロイする。
    - `railway vars` を使用して、データベース接続情報や JWT シークレットなどの環境変数を設定する。
- **データベース構築 (Railway PostgreSQL):**
    - Railway 上で PostgreSQL インスタンスをプロビジョニングする。
    - デプロイプロセスの中で Alembic マイグレーションを自動実行し、テーブルを作成する。
    - `seed_assets.py` を実行し、初期資産データを自動的に投入する。
- **環境間連携:**
    - フロントエンドから Railway 上のバックエンド API への通信が正常に行えるように設定する（CORS 設定を含む）。
    - 認証（Supabase Auth）が本番環境のドメインで正しく動作するように設定する。

## 3. Non-Functional Requirements (非機能要件)
- **セキュリティ:** CLI ツールを使用することで、機密情報（API キーやパスワード）をソースコードや Git に含めず、安全に管理する。
- **再現性:** デプロイコマンドにより、データベースのスキーマ構築とデータ投入が自動で行われるようにし、手動操作を最小限にする。
- **可用性:** 各プラットフォームのデフォルト設定を利用し、基本的な死活監視が機能するようにする。

## 4. Acceptance Criteria (受け入れ条件)
- [ ] Vercel のデフォルト URL（`xxx.vercel.app`）でフロントエンドにアクセスできること。
- [ ] Railway のデフォルト URL（`xxx.railway.app`）でバックエンド API にアクセスできること。
- [ ] フロントエンドからバックエンドへの API リクエストが成功し、シミュレーション結果が表示されること。
- [ ] Supabase を使用したログイン・サインアップが正常に動作すること。
- [ ] Railway 上のデータベースに初期資産データが正しく投入されていること。

## 5. Out of Scope (対象外)
- カスタムドメイン（独自ドメイン）の設定。
- 高度な CI/CD パイプライン（GitHub Actions 等）の構築（プラットフォーム標準の連携機能を使用する）。
- パフォーマンステストや大規模な負荷試験。
