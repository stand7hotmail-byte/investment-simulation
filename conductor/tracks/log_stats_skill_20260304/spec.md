# Specification: Log Analysis and Statistics Display Skill

## Overview
Gemini CLIの実行ログ（JSON形式）を解析し、スキルやツールの使用統計、およびエラー発生頻度をターミナル上にテーブル形式で表示する新しいスキルを作成する。これにより、ユーザーは自身の開発プロセスにおけるAIツールの活用状況を定量的に把握できる。

## User Personas
- **AI Agent Developer**: 自身の開発したスキルやMCPサーバーがどの程度、またどのように活用されているかを知りたい開発者。
- **Power User**: 自身のトークン消費やツール利用の傾向を把握し、効率化を図りたいユーザー。

## Functional Requirements

### 1. ログ解析エンジン
- **ソース指定**: プロジェクトルートにある特定のログファイル（デフォルト: `dev_server.log`）を読み込む。
- **フォーマット対応**: JSON形式（Structured Logging）のログをパースする。
- **セッション抽出**: ログ内の最新のセッション（最後の起動から現在まで）のみを抽出して集計対象とする。

### 2. 統計集計項目
- **スキル呼び出し統計**: 各スキルの起動回数を集計する。
- **MCPツール呼び出し統計**: MCPサーバー経由で呼び出された各ツールの実行回数を集計する。
- **エラー統計**: 実行中にエラー（失敗）を返したツールの種類とその頻度を集計する。

### 3. 表示機能
- **出力形式**: CLIターミナル上でのプレーンテキスト形式。
- **レイアウト**: 視認性の高いテーブル（表）形式で出力する。
- **表示内容**: 各項目名、実行回数、成功/失敗の比率（可能であれば）を表示する。

## Non-Functional Requirements
- **パフォーマンス**: ログファイルの読み込みと集計は数秒以内に完了すること。
- **堅牢性**: ログの一部が破損している、または予期しない形式であっても、スキル全体がクラッシュせず、解析可能な範囲で結果を表示すること。

## Acceptance Criteria
- [ ] 指定したログファイルから最新セッションのデータを正しく抽出できること。
- [ ] スキル、MCPツール、エラーの各集計値が実際のログ内容と一致すること。
- [ ] ターミナル上で崩れのないテーブル形式で統計が表示されること。
- [ ] ログファイルが存在しない、または空の場合に適切なエラーメッセージが表示されること。

## Out of Scope
- 過去の全セッションにわたる長期的なトレンド分析（今回は最新セッションのみ）。
- 外部データベースへの統計データの保存。
- グラフやHTMLレポートの生成。
