# Track Specification - Advanced Portfolio Analytics

## Overview
既存のポートフォリオ管理・シミュレーション機能を拡張し、より実戦的な投資判断を支援するための3つの高度な分析機能（リバランス提案、歴史的ストレステスト、動的配当シミュレーション）を導入します。これらの機能は、各ポートフォリオの「詳細画面」に新しい分析タブとして統合されます。

## Objectives
- **Actionability**: 最適な資産配分へ戻すための具体的なリバランス指針を提供。
- **Risk Awareness**: 過去の暴落シナリオを適用し、ポートフォリオの最大下落耐性を可視化。
- **Income Clarity**: 過去の配当実績に基づき、将来得られる見込みのインカムゲインを予測。

## Functional Requirements
### 1. Smart Rebalancing (比率差分表示)
- 選択された「目標ポートフォリオ（効率的フロンティア上の点など）」と、現在のポートフォリオの各資産比率を比較。
- 各資産について、目標に対してどれだけ不足/超過しているか（±%）を視覚的に表示。

### 2. Market Stress Test (歴史的暴落の再現)
- 以下の期間の市場データを抽出し、現在のポートフォリオ構成を当てはめた場合の騰落率を計算：
    - リーマンショック (2008年9月〜)
    - コロナショック (2020年2月〜)
    - ITバブル崩壊 (2000年3月〜)
- 最大ドローダウン（資産の最大下落幅）と、回復までにかかった期間を可視化。

### 3. Dynamic Income Simulation (配当シミュレーション)
- `yfinance` を通じて各資産の過去の配当実績（Dividends）を取得。
- 過去1年間の配当利回り、および過去の増配傾向を考慮した将来のインカムゲイン推移を表示。
- キャピタルゲイン（値上がり益）とインカムゲインのトータルリターンを積み上げグラフで表示。

### 4. Portfolio Analysis UI
- ポートフォリオ詳細ページに「分析（Analytics）」タブを新設。
- リバランス、ストレス、インカムの各セクションをダッシュボード形式で配置。

## Technical Requirements
- **Backend**:
    - `yfinance` の `actions` 属性から配当データを取得するロジックの追加。
    - ストレステスト用の歴史的騰落率計算エンジンを `simulation.py` に追加。
- **Frontend**:
    - `recharts` または `plotly.js` を使用した積み上げグラフおよび下落チャートの描画。
    - 各分析結果を表示するための新しいサブコンポーネント群。

## Acceptance Criteria
- ポートフォリオ詳細画面から、現在の構成に対するリバランス案が表示されること。
- 指定した3つの暴落シナリオにおいて、ポートフォリオがどう動くかのシミュレーションが表示されること。
- 過去の配当実績に基づいた年間の見込み配当額が算出されること。
- 全ての新しいAPIエンドポイントに対してテストが記述されていること。
