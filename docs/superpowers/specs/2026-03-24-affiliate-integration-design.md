# デザインドキュメント: 証券会社アフィリエイト統合

**日付:** 2026-03-24
**ステータス:** 草案
**トピック:** 証券会社アフィリエイト連携による収益化

## 1. 背景・目的
投資シミュレーション（効率的フロンティア、モンテカルロ）を利用し、投資意欲が高まったユーザーに対し、最適な証券会社の口座開設を促すことでアフィリエイト収入を得る。
日本国内および海外ユーザーの両方をターゲットとし、地域に応じた最適な選択肢を自動的に提示する。

## 2. ユーザー体験 (UX)
- ユーザーがシミュレーションを完了し、結果グラフ（Plotly.js）を確認した直後、グラフのすぐ下に「このポートフォリオを実現するためのおすすめ証券会社」セクションが表示される。
- 地域（日本/グローバル）は、サーバーサイドでIPアドレス等から自動的に判別され、ユーザーの手間なく最適なカードが表示される。
- クリーンで信頼感のあるカード形式を採用し、各社の強みを簡潔に比較できる。

## 3. アーキテクチャと実装詳細 (バックエンド管理・柔軟型)

### 3.1 データベース設計 (PostgreSQL)
新しいテーブル `affiliate_brokers` を作成する。

| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | UUID/INT | プライマリキー |
| `name` | VARCHAR | 証券会社名 (例: "SBI証券") |
| `region` | VARCHAR | ターゲット地域 ("JP" or "GLOBAL") |
| `description` | JSONB | 強みの箇条書き (例: `["手数料無料", "Vポイント貯まる"]`) |
| `cta_text` | VARCHAR | ボタンラベル (例: "口座開設はこちら") |
| `affiliate_url` | TEXT | アフィリエイトURL |
| `logo_url` | TEXT | (オプション) 会社ロゴのURL |
| `is_active` | BOOLEAN | 有効/無効の切り替えフラグ |
| `priority` | INTEGER | 表示の優先順位（降順） |
| `created_at` | TIMESTAMP | 作成日時 |

### 3.2 APIエンドポイント (FastAPI)
- `GET /api/v1/affiliates/recommendations`
  - **ロジック:** 
    1. リクエストヘッダー（`X-Forwarded-For`）からクライアントIPを抽出。
    2. 国コードを判定（初期実装ではシンプルなIP-to-Countryライブラリまたは外部APIを利用、判定不能時はデフォルトでGLOBAL）。
    3. `region` が一致し、かつ `is_active=True` のレコードを `priority` 順に取得。
    4. フロントエンドにリスト形式で返す。

## 4. フロントエンド設計 (Next.js/TypeScript)

### 4.1 コンポーネント
- **`AffiliateBrokerCard.tsx`**: 単体カード。Tailwind CSSを用いたモダンなデザイン。
- **`AffiliateSection.tsx`**: 
  - `fetchApi` を使用してエンドポイントからデータを取得。
  - スケルトンローダー（ロード中の表示）の実装。
  - データが0件の場合はセクション自体を非表示にする。

### 4.2 配置場所
- `frontend/src/app/simulation/efficient-frontier/results/page.tsx`
- `frontend/src/app/simulation/accumulation/results/page.tsx`
- (今後追加されるシミュレーション結果ページ)

## 5. 運用と拡張
- **リンク更新:** アフィリエイトリンクの変更やキャンペーン対応は、DBのレコード更新のみで完結し、コード修正や再デプロイを不要とする。
- **ABテスト:** 将来的には、クリック率を計測し、`priority` を動的に変更することで収益を最大化するロジックの追加を検討する。

## 6. テスト計画
- 日本IPと海外IPそれぞれで正しいデータが返されるかの単体テスト。
- APIがエラー（500等）を返した場合に、フロントエンドの表示が崩れないことの確認。
- リンククリックの計測（Google Analytics等のイベント発火）の確認。
